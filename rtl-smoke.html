Read the control column. A neutral character only strands itself when it sits at an edge of the run with nothing strong to anchor it, so the rows do not all misbehave — and saying which ones should is the difference between a test and a decoration: rows 1, 4, 5 are expected to differ (leading #, a numeral-plus-unit with no strong letter in it at all, and a trailing …). Rows 2, 3, 6 are expected to look the same in both columns — their ellipsis sits between alphanumerics and their leading d/0x anchors the run, so they are safe with or without the rule. They are here as the negative control: a fixture where everything fails is as untrustworthy as one where everything passes. If rows 1, 4 or 5 stop differing, this fixture has died — fix it before believing a pass. In every case the DOM value is untouched and only the display is wrong, which is why this class of bug clears code review and then fails a human checking an identifier by eye.
